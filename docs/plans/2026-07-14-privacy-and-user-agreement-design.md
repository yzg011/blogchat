# 隐私政策与用户协议弹窗设计

## 背景

现有隐私政策沿用通用互联网产品模板，包含账户注册、定位、广告投放及跨境传输等本站未提供的能力，不符合个人博客的实际运行方式。

本站在页脚通过弹窗展示隐私政策。用户协议也采用相同交互，以保持访问路径简洁且不新增页面路由。

## 目标

- 重写隐私政策，使其准确反映个人博客的数据处理活动。
- 明确说明实际使用的第三方服务及其用途。
- 新增用户协议，明确文章评论与留言板的发布规则、管理措施、免责声明和未成年人条款。
- 在页脚新增“用户协议”入口，并以弹窗形式显示。

## 真实数据处理范围

| 服务或能力 | 处理的信息 | 用途 |
| --- | --- | --- |
| Umami 统计 | 页面访问、设备与浏览器相关信息、来源和访问行为 | 了解网站使用情况和改进内容 |
| Waline 评论 | 昵称、邮箱或站点链接（如主动填写）、评论内容、IP 地址等必要信息 | 发布和管理评论、防垃圾与服务安全 |
| Cloudflare Workers 与 KV | 留言昵称、留言内容、创建时间、投票数据，以及用于限流的 IP 地址 | 提供留言板、保存数据并防止滥用 |
| unpkg 表情资源 | 发起评论页面时的资源请求信息 | 加载 Waline 评论表情 |

不声明账户注册、定位、广告投放、个性化推荐、支付、跨境传输或主动出售个人信息等本站未提供的能力。

## 内容设计

### 隐私政策

内容使用面向个人博客的简明表述，并包含以下章节：

1. 适用范围与站点说明。
2. 信息收集与使用：访问统计、评论、留言板与反滥用处理。
3. 第三方服务：逐项说明 Umami、Waline、Cloudflare 与 unpkg 的角色及跳转至其隐私说明的链接。
4. Cookie 及本地存储的使用：主题、显示偏好、播放器与交互偏好等浏览器本地数据；说明如何通过浏览器清理或限制。
5. 信息公开、保存与安全：评论和留言的公开性、删除范围、安全措施及技术风险。
6. 你的权利：查询、更正、删除或就处理提出疑问的联系方式；说明评论/留言删除时需要提供可核验信息。
7. 未成年人条款、政策更新和联系方式。

更新日期与生效日期均保留为 2026 年 5 月 19 日。

### 用户协议

用户协议使用独立 Markdown 内容，包含以下章节：

1. 适用范围与接受方式。
2. 评论及留言规则：禁止违反中华人民共和国法律法规、侵犯他人权益、攻击辱骂骚扰威胁歧视、垃圾广告或刷屏、诈骗钓鱼及恶意软件、绕过审核或封禁等行为。
3. 内容与账号管理：管理员可不另行通知删除违规内容、限制或封禁账号，并保留依法配合处理的权利。
4. 知识产权与内容授权：用户应保证发布内容拥有合法权利；为展示和管理评论或留言而授予本站必要的非独占许可。
5. 免责声明：博客内容仅供交流参考；外部链接、用户发布内容及服务中断不构成承诺或担保。
6. 未成年人条款、协议变更、争议处理与联系方式。

## 实现结构

新增 `src/content/spec/user-agreement.md` 作为用户协议的唯一内容来源。新增 `UserAgreementModal.astro`，其 DOM、关闭行为与生命周期初始化遵循 `PrivacyModal.astro` 的现有模式。

`Layout.astro` 挂载新弹窗，使其在 Swup 页面切换时保持可用。`Footer.astro` 在 RSS 与隐私政策同一行新增“用户协议”按钮，调用全局的弹窗打开方法。对应样式放入既有的隐私弹窗样式文件或同层样式文件，并由 `main.css` 导入，保持主题与移动端表现。

## 交互与异常处理

- 点击页脚“隐私政策”或“用户协议”分别只打开对应弹窗。
- 点击关闭按钮、遮罩或按 Escape 关闭当前弹窗。
- 内容集合缺失时构建失败，避免上线一个没有正文的空弹窗。
- 原有隐私弹窗的打开和关闭行为不得回归。

## 验证

先为用户协议内容与页脚入口编写可失败的测试或构建断言，确认新增内容和触发器在实现前不存在。实现后运行针对性测试、`pnpm check` 和生产构建；在桌面与移动视口核查两个页脚入口、弹窗内容与关闭操作。
